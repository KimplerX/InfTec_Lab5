Ця база даних (test_shop) являє собою типову, хоча й трохи надмірну, схему для невеликого інтернет-магазину. Вона сфокусована на трьох основних групах сутностей: клієнтах, товарах та транзакціях.

1. Управління Клієнтами
База даних містить дві дуже схожі таблиці для користувачів (customer та customers), що свідчить про можливе перейменування чи рефакторинг в історії проекту. Кожна сутність зберігає особисті дані (ім'я, прізвище, місто), контактну інформацію (email, телефон) та хешований пароль. Важливою є наявність поля admin_role, що дозволяє розрізняти звичайних користувачів та адміністраторів магазину.

2. Товари та Каталог
Товари зберігаються у таблиці products. Кожен товар має унікальний артикул (sku), назву, ціну (price) та інформацію про залишок на складі (qty).
